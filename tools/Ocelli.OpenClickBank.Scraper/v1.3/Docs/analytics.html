Analytics Service Description
The following methods are available at the servlet path /rest/1.3/analytics
GET/1.3/analytics/{role}/subscription/details/compthirty
Returns a list of subscriptions completing in the next 30 days.

Return Type:A SubscriptionDetailResults

Request Parameters :

Name Required Description
account true The account nickname
orderBy false Valid options include:

Value SubscriptionDetailResultRow
Equivalent
RECEIPT receipt
PURCHASE_DATE purchaseDate
SUB_END_DATE subEndDate
SUB_CANCEL_DATE subCancelDate
NEXT_PAYMENT_DATE nextPaymentDate
SUB_VALUE subValue
STATUS status
ITEM_NUMBER itemNo
PROCESSED_PAYMENTS_COUNT processedPaymentsCount
FUTURE_PAYMENTS_COUNT futurePaymentsCount
REFUND_COUNT refundCount
REFUND_AMOUNT refundAmount
CHARGEBACK_COUNT chargebackCount
CHARGEBACK_AMOUNT chargebackAmount
PUB_NICK_NAME pubNickName
AFFILIATE_NICK_NAME affNickName
CUSTOMER_FIRST_NAME* customerFirstName
CUSTOMER_LAST_NAME* customerLastName
CUSTOMER_DISPLAY_NAME* customerDisplayName
CUSTOMER_EMAIL* email
DURATION duration
INITIAL_SALE_AMOUNT initialSaleAmount
INITIAL_SALE_COUNT initialSaleCount
REBILL_SALE_AMOUNT rebillSaleAmount
REBILL_SALE_COUNT rebillSaleCount
*Customer details are only available to vendors
sortDirection false The order in which the sorted results are returned:
  • ASC (ascending - default)
  • DESC (descending)

Supported Output Types : application/xml, application/json

Required Roles : api_analytics_client, HAS_DEVELOPER_KEY

GET/1.3/analytics/{role}/subscription/details/startdate
Returns a list of subscriptions where the subscription start date is between (inclusive) the startDate and endDate parameters.

Return Type:A SubscriptionDetailResults

Request Parameters :

Name Required Description
account true The account nickname
orderBy false See compthirty orderBy
sortDirection false The order in which the sorted results are returned:
  • ASC (ascending - default)
  • DESC (descending)
startDate true The earliest subscription start date the result list will contain. Date Format: yyyy-MM-dd
endDate true The latest subscription start date the result list will contain. Date Format: yyyy-MM-dd

Supported Output Types : application/xml, application/json

Required Roles : api_analytics_client, HAS_DEVELOPER_KEY

GET/1.3/analytics/{role}/subscription/trends
Returns statistical summations of data for subscriptions

Valid roles include:
  • AFFILIATE
  • VENDOR

Return Type:A SubscriptionTrendsData

Head Parameters :

Name Required Description
Page false The page number of the results (default is page 1)

Request Parameters :

Name Required Description
account true The account nickname
groupBy false You may group by business date by passing DATE as the value
productId false The product id to report on, multiple parameter/value pairs may be passed
startDate true The start date (inclusive) of the time frame to report on - format is yyyy-MM-dd.
endDate true The end date (inclusive) of the time frame to report on - format is yyyy-MM-dd.

Supported Output Types : application/xml, application/json

Required Roles : api_analytics_client, HAS_DEVELOPER_KEY
GET/1.3/analytics/status

Return the status & last update time of the API.

The possible statuses are:

  • GREEN (normal operation)
  • YELLOW (delayed data)
  • RED (severely delayed data or critical problem)

Return Type:An AnalyticsStatus

Supported Output Types : application/xml, application/json, text/csv

Required Roles : api_analytics_client, HAS_DEVELOPER_KEY
